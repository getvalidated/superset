## Sentry webhook → agent on the bug

When Sentry fires an alert, automatically:

1. Create a task with the error details.
2. Spin up a workspace on a developer's machine.
3. Dispatch an agent into that workspace to investigate.

A minimal Hono handler:

```ts title="sentry-webhook.ts"
import { Hono } from 'hono';
import Superset from '@superset_sh/sdk';

const app = new Hono();
const client = new Superset();
// reads SUPERSET_API_KEY + SUPERSET_ORGANIZATION_ID from env

const TARGET_HOST_ID = process.env.SUPERSET_TARGET_HOST_ID!;  // hosts.list() to find yours
const PROJECT_ID = process.env.SUPERSET_PROJECT_ID!;          // projects.list() to find yours

app.post('/webhooks/sentry', async (c) => {
  const event = await c.req.json();
  const { title, web_url: sentryUrl, culprit } = event.data.issue;
  const slug = `sentry-${event.data.issue.short_id.toLowerCase()}`;

  // 1. File a task so it shows up in the dashboard
  const task = await client.tasks.create({
    title: `[Sentry] ${title}`,
    description: `Triggered by ${culprit}\n\nSentry: ${sentryUrl}`,
    priority: 'high',
    labels: ['sentry', 'bug'],
  });

  // 2. Spin up a worktree on a developer machine, branch off the issue slug
  const workspace = await client.workspaces.create({
    hostId: TARGET_HOST_ID,
    projectId: PROJECT_ID,
    name: slug,
    branch: slug,
  });

  // 3. Create a one-off automation pinned to the new workspace + host,
  //    schedule it for the past so it dispatches immediately, then run it.
  const automation = await client.automations.create({
    name: `Investigate ${event.data.issue.short_id}`,
    prompt: [
      `A Sentry alert just fired:`,
      `  Title: ${title}`,
      `  Culprit: ${culprit}`,
      `  URL: ${sentryUrl}`,
      ``,
      `Reproduce the failure, fix it, push a branch, and update task ${task.id}`,
      `with what you found.`,
    ].join('\n'),
    agentConfig: { id: 'claude', kind: 'terminal' },
    targetHostId: TARGET_HOST_ID,
    v2WorkspaceId: workspace.id,
    rrule: 'FREQ=YEARLY;BYMONTH=12;BYMONTHDAY=31',  // never re-runs
    timezone: 'UTC',
  });

  await client.automations.run(automation.id);

  return c.json({ ok: true, taskId: task.id, workspaceId: workspace.id });
});

export default app;
```

What just happened, end-to-end:

- The task lives in the cloud and appears in the dashboard for everyone.
- The workspace is a real `git worktree` on the target machine, ready for a human to take over if the agent gets stuck.
- The agent runs in that workspace's terminal, with full access to the codebase and whatever MCP servers the host has configured.

## Daily triage report

A scheduled job that posts your team's open high-priority tasks to Slack:

```ts title="daily-triage.ts"
import Superset from '@superset_sh/sdk';

const client = new Superset();

const urgent = await client.tasks.list({ priority: 'urgent', limit: 50 });
const high   = await client.tasks.list({ priority: 'high', limit: 50 });

const lines = [
  `*Triage — ${new Date().toLocaleDateString()}*`,
  ``,
  `*Urgent (${urgent.length})*`,
  ...urgent.map((t) => `• ${t.slug} — ${t.title} ${t.assigneeName ? `(${t.assigneeName})` : '_unassigned_'}`),
  ``,
  `*High (${high.length})*`,
  ...high.map((t) => `• ${t.slug} — ${t.title} ${t.assigneeName ? `(${t.assigneeName})` : '_unassigned_'}`),
];

await fetch(process.env.SLACK_WEBHOOK_URL!, {
  method: 'POST',
  headers: { 'content-type': 'application/json' },
  body: JSON.stringify({ text: lines.join('\n') }),
});
```

Wire this up to Vercel Cron, GitHub Actions, or whatever scheduler you have. The denormalized fields on `TaskListItem` (`assigneeName`, `statusName`, …) mean you don't need follow-up calls for display.

## Bulk reassign on a label

A one-off script you can run from your terminal:

```ts title="reassign.ts"
import Superset from '@superset_sh/sdk';

const client = new Superset();

const NEW_OWNER_ID = '<uuid>';

const stale = await client.tasks.list({ search: 'flaky', limit: 500 });

for (const t of stale) {
  await client.tasks.update({ id: t.id, assigneeId: NEW_OWNER_ID });
  console.log(`reassigned ${t.slug} → ${NEW_OWNER_ID}`);
}
```

`tasks.list` filters AND-combine, so layer in `priority`, `statusId`, `creatorMe`, etc. to narrow the query.

## On-demand agent dispatch from your tools

If you've already created a recurring automation in the dashboard, the SDK can fire it off-schedule from anywhere — a Slack slash command, a CI job, an internal admin panel:

```ts
const run = await client.automations.run('<automation-id>');
console.log(`dispatched run ${run.id} to host ${run.hostId}`);
```

The automation's pinned host has to be online and tunneling. The SDK will surface a `503 Host not connected` error if it isn't, so you can retry later or page someone.
