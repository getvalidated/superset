# Workspaces Create — Test Plan & V1 vs Current Comparison

Goal: confirm the unified `workspaces.create` procedure on this branch
matches (or intentionally diverges from) the v1 behavior on `main`. Run
each case against a real host service and inspect results in the host
SQLite + cloud `v2_workspaces` + on-disk worktrees.

## V1 (origin/main, before this PR)

Two server procedures; renderer dispatched to one or the other based on
intent (Compose vs. Check-out / PR row in the picker):

- **`workspaceCreation.create`** — *always creates a new branch* off a
  resolved start point. Inputs: `projectId`, `names.{workspaceName,
  branchName, workspaceNameWasAutoGenerated?}`, `composer.{prompt?,
  baseBranch?, baseBranchSource?, runSetupScript?}`, `linkedContext`,
  `pendingId`. Behavior:
  1. `clearProgress` finally guard, `setProgress` step pings.
  2. `requireLocalProject` + `ensureMainWorkspace`.
  3. `deduplicateBranchName(names.branchName, existingBranches)` —
     appends `-2`, `-3`… on collision.
  4. `safeResolveWorktreePath` + `mkdirSync` parent dir.
  5. Resolve start point: prefer the picker hint
     (`baseBranchSource: "local" | "remote-tracking"` →
     `buildStartPointFromHint`), else probe via `resolveStartPoint`.
     If the start point is the *local default branch*, swap to its
     upstream remote ref so we fork from the real tip. Fetch the
     remote ref if applicable.
  6. `worktree add --no-track -b <branch> <path> <startPoint>`. Always
     creates a brand-new local branch.
  7. `enablePushAutoSetupRemote` and write `branch.<name>.base` config
     (skipped for `head` start point).
  8. `host.ensure` → `v2Workspace.create` → insert local `workspaces`
     row (rolling back the worktree on each failure, deleting the
     cloud row if the local insert fails).
  9. Fire-and-forget AI rename when `composer.prompt` set AND
     `names.workspaceNameWasAutoGenerated !== false`.
  10. If `composer.runSetupScript`, spawn `bash .superset/setup.sh`
     terminal (if the file exists in the worktree).
  11. Returns `{ workspace, terminals, warnings }`.

- **`workspaceCreation.checkout`** — *checks out an existing branch or
  a PR*. Inputs: `projectId`, `workspaceName`, `pendingId`, exactly one
  of `branch` or `pr`, `composer.{prompt?, baseBranch?, runSetupScript?}`,
  `linkedContext`. Behavior:
  - **PR path**: derive local branch name from PR; idempotency check by
    branch (`existing → return alreadyExists`); detect *pre-existing
    local branch outside Superset* and **warn** (not block) — `gh pr
    checkout --force` would silently reset it; `worktree add --detach`
    then `gh pr checkout --branch X --force` inside; `enablePush…`;
    pass `pr.state !== "open"` and pre-existing-branch warnings into
    `finishCheckout`.
  - **Branch path**: `resolveRef` → if `null | "head" | "tag"` →
    `BAD_REQUEST` ("Branch X does not exist locally or on origin");
    fetch if remote-tracking; `worktree add --track -b <branch> <path>
    origin/<name>` for remote-only or `worktree add <path> <branch>`
    for local; `enablePush…`.
  - Both fall through to `finishCheckout`: write `branch.<name>.base`
    from `composer.baseBranch` (no smart resolve); host.ensure +
    cloud + local insert + optional setup terminal.

- Renderer (in `useCheckoutDashboardWorkspace` / similar) **separately**
  fired agent launches via host service after the workspace existed.
  Server had no `agents` field.

- v1 schemas in `workspace-creation/schemas.ts` carried the picker
  hint (`baseBranchSource`), the `runSetupScript` flag, the progress
  pendingId, and `linkedContext`.

## Current (this branch)

Single procedure **`workspaces.create`** with input:

```
{
  projectId, name,
  branch?: string, pr?: number,   // exactly one of these
  baseBranch?: string,
  taskIds?: uuid[], autogenerateName?: boolean,
  agents?: { agent, prompt, attachmentIds? }[],
  id?: uuid                       // explicit id for idempotent retry
}
```

Behavior:

1. `requireLocalProject` + `ensureMainWorkspace`.
2. **PR path** (`input.pr`): `gh pr view --json …` → `prMetadata`,
   `derivePrLocalBranchName(prMetadata)`. Idempotency: existing local
   `workspaces` row with that branch → return `alreadyExists: true` (no
   reuse warnings). Else **`localBranchExists`** check rejects with
   **CONFLICT** if a local branch with the same derived name exists
   outside Superset (does *not* warn-and-overwrite like v1). Then
   detached worktree → `gh pr checkout` → autoSetupRemote → register
   cloud+local → `recordBaseBranchConfig(prMetadata.baseRefName)`. **No
   warnings** for closed/merged PR or for pre-existing local branch.
3. **Branch path**: `resolvedBranch = input.branch.trim()` (no dedup).
   Idempotency check by branch: existing → `alreadyExists: true`.
   Otherwise:
   - **Adopt check**: if `getWorktreeBranchAtPath(git, worktreePath) ===
     resolvedBranch` (a worktree already at the standard path with
     this branch checked out), skip `git worktree add` and just
     register. New: didn't exist in v1.
   - `planBranchSource(git, branch, baseBranch)`:
     - `resolveRef(git, branch)`: if it resolves local/remote-tracking
       → `usedExistingBranch: true`; if tag → BAD_REQUEST; else build
       start point.
     - Start point: `resolveStartPoint(git, baseBranch)` (defaults to
       repo default if no `baseBranch`). Same upstream-swap heuristic
       as v1 *probe* path. **No picker-hint shortcut** — the
       `baseBranchSource` field was removed.
     - Fetch if remote-tracking.
   - `addBranchWorktree(plan)`:
     - `usedExistingBranch=true` + remote-tracking → `worktree add
       --track -b <branch> <path> <remote>/<name>`
     - `usedExistingBranch=true` + local → `worktree add <path>
       <branch>` (or `worktree add <path> HEAD` if start point is
       head)
     - `usedExistingBranch=false` → `worktree add --no-track -b
       <branch> <path> <startPointArg>`
   - `enablePushAutoSetupRemote`.
   - Write `branch.<name>.base` config **only** when
     `!plan.usedExistingBranch && plan.startPoint.kind !== "head"`. v1
     wrote it from `composer.baseBranch` regardless on the
     existing-branch path.
   - `registerCloudAndLocal` (host.ensure → v2Workspace.create with
     optional explicit `id` + `taskIds` → local insert; rolls back on
     failure as v1 did).
4. After register, when `!alreadyExists`:
   - `startSetupTerminalIfPresent` always (no `runSetupScript` flag —
     setup runs whenever `.superset/setup.sh` exists in the worktree).
   - If `input.autogenerateName === true` AND there's a non-empty
     prompt on `agents[0]` → fire-and-forget `applyAiWorkspaceRename`.
5. **`dispatchSugarAgents`** for every entry in `input.agents`. Each
   maps to `runAgentInWorkspace(ctx, …)` which resolves the agent
   config from `host_agent_configs`, builds the shell command, and
   spawns a terminal session. Per-agent failure is captured as
   `{ ok: false, error }` rather than failing the whole call.
6. Returns `{ workspace, terminals, agents, alreadyExists }`.

## Behavior diff matrix

| # | Aspect | V1 | Current | Notes |
|---|---|---|---|---|
| 1 | Branch dedup on collision | yes (`deduplicateBranchName`) | **no** | Risk: friendly-random or AI-generated name colliding with an orphan local branch silently bounces into the existing-branch path. Acceptable today because the renderer drives uniqueness; flag as future-fix candidate. |
| 2 | Idempotency by `(project, branch)` | branch path: no; PR path: yes | yes both paths | Aligned. |
| 3 | Worktree adoption (existing dir at standard path) | no | yes (branch path) | New. Matches v1→v2 migration semantics. |
| 4 | Tag rejection | yes (BAD_REQUEST) | yes (BAD_REQUEST) | Same. |
| 5 | Picker hint (`baseBranchSource`) | yes | **no** — always probes | Re-resolves locally; risk of stale cached `refs/remotes/origin/<name>` winning over the picker's intent. |
| 6 | Local-default-branch → upstream swap | yes | yes | Same. |
| 7 | `--quiet` rev-parse bug | present (succeeds with empty stdout, treated as exists) | **fixed** | Fixed in this PR; root cause of "create succeeds but uses wrong start point". |
| 8 | PR + pre-existing local branch | warn-and-`--force` (resets to PR HEAD, recoverable via reflog) | **CONFLICT** error | Stricter. Trade-off: safer but a user with a stray same-named branch can't `gh pr checkout` from the modal anymore. |
| 9 | PR closed/merged warning | yes (`extraWarnings`) | **no** | UX regression — verify the renderer surfaces this differently or accept. |
| 10 | Setup terminal | gated on `composer.runSetupScript` | **always** if `.superset/setup.sh` exists | Renderer no longer toggles. |
| 11 | Agent dispatch atomicity | renderer-driven, separate calls | server-inlined via `agents` array | New PR1 capability. |
| 12 | Agent dispatch errors | renderer surfaced per-call errors | per-entry `{ ok, ... }`, never fails create | New behavior. |
| 13 | Agent dispatch with empty `host_agent_configs` | n/a (no server agents) | **silent failure** — every agent returns `{ ok: false, error: NOT_FOUND }`, no terminals spawn | Filed task `host-agent-configs-aren-t-seeded-until-user-visits` for lazy-seed bug. |
| 14 | TaskIds linking | not part of create | yes (atomic insert into `workspace_tasks` in the same v2Workspace.create txn) | New. |
| 15 | Explicit `id` for retry | no | yes (idempotent retry surfaces same workspace) | New. |
| 16 | AI rename gate | `workspaceNameWasAutoGenerated !== false` (default true) | `autogenerateName === true` (default false) | Polarity flipped — old behavior was "rename unless told not to"; new is "rename only when told to". |
| 17 | Progress pings (`setProgress`) | yes (renderer polled) | **no** (renderer in-flight store handles UX) | Acceptable — `useWorkspaceCreates` covers it. |
| 18 | base-branch config write on existing-branch path | yes (writes `composer.baseBranch`) | **no** (only writes on new-branch path) | Verify Changes-tab still gets a base for adopted branches via other means, or accept. |

## Input × pre-existing state matrix

The full enumeration of `workspaces.create` cases. **Inputs** are
what the caller sends. **State** is what already exists on disk /
in DB before the call. **Outcome** is what should happen.

After this PR's refactor (server-generated names + per-side AI
rename gate), the inputs collapse to:

```
{
  projectId,
  name?,         // user-typed workspace title; absent = AI rename it
  branch?,       // user-typed git branch; absent = generate friendly + AI rename
  pr?,           // PR number (mutually exclusive with branch)
  baseBranch?,   // start-point hint for new branches
  taskIds?,
  agents?,       // [{ agent, prompt, attachmentIds? }] — sugar for create+dispatch
  id?            // explicit UUID for idempotent retry
}
```

### A. Validation / early reject

| # | Inputs | State | Outcome |
|---|---|---|---|
| A1 | `branch` set AND `pr` set | — | Zod refine: BAD_REQUEST `"branch and pr cannot both be set"` |
| A2 | `projectId` not in host's `projects` table | — | `requireLocalProject` throws (NOT_FOUND-style); no rollback needed (no side effects yet) |
| A3 | `taskIds` includes a uuid not in `tasks` table | task missing in cloud | Cloud `v2Workspace.create` rejects BAD_REQUEST (taskId not found); worktree rolled back |
| A4 | `taskIds` from a different organization | task in another org | Cloud rejects FORBIDDEN; worktree rolled back |
| A5 | `id` collides with an existing v2 workspace in **another** org | cloud row exists in different org | Cloud rejects CONFLICT (workspace id already in use); worktree rolled back |
| A6 | `id` matches an existing v2 workspace in **same** org | cloud row exists | Cloud transaction returns existing row → server treats as alreadyExists path (idempotent retry) |

#### C.2 — `branch` not typed

Server generates `friendly-random` (e.g., `nosy-puck`) and dedupes against existing local branches.

### D. Failure / rollback paths (orthogonal to the input matrix)

| # | Failure point | Outcome |
|---|---|---|
| D1 | Friendly-name dedup loop exhausts (>10000 collisions) | Falls back to `<base>-<unix-ms-base36>`. Effectively never collides. |
| D2 | `git worktree add` fails (e.g. path occupied) | `worktree remove --force` rollback (no-op when add failed). CONFLICT bubbled. No cloud or local rows. |
| D3 | `gh pr checkout` fails after detached worktree added | `worktree remove --force` rollback. INTERNAL_SERVER_ERROR. No rows. |
| D4 | `host.ensure` fails | Worktree rolled back. INTERNAL_SERVER_ERROR. No cloud or local rows. |
| D5 | `v2Workspace.create` cloud rejects | Worktree rolled back. Cloud error bubbled. No local row. |
| D6 | Local sqlite insert fails (foreign key, disk full, etc.) | Worktree rolled back AND cloud row deleted via `v2Workspace.delete`. INTERNAL_SERVER_ERROR. |
| D7 | Setup terminal spawn fails | Workspace + cloud row REMAIN. Setup-terminal warning logged. Create call returns success — terminal failure is independent. |
| D8 | Agent dispatch fails (e.g. empty `host_agent_configs`) | Workspace + cloud row REMAIN. `agents` array in response has `{ ok: false, error }` per failed entry. Create call returns success. |
| D9 | AI rename LLM call fails / model unavailable | Workspace + cloud row REMAIN with the original (typed-or-friendly) name. AI rename is fire-and-forget; failure logged but not bubbled. |
| D10 | AI rename's `git branch -m` fails (e.g. branch lock) | LLM call succeeded but git rename didn't fire. Title still applied if `renameTitle`. Branch unchanged. |
| D11 | AI rename's cloud `updateNameFromHost` fails after git rename | Git rename rolled back via reverse `git branch -m`. Cloud + local DB stay in sync. |

### E. AI rename × user-typed matrix (post-PR1 behavior)

After this PR, AI rename runs whenever the user didn't supply a side
AND there's a non-empty agent prompt. Both flags are computed at the
call site:

```
renameTitle  = input.name === undefined
renameBranch = input.branch === undefined
```

PR path always sets both to false (skips AI rename — PR title is meaningful).

| `branch` | `name` | `prompt` | AI rename behavior |
|---|---|---|---|
| typed | typed | any | Skipped — both supplied |
| typed | undefined | empty | Skipped — no prompt to drive generation |
| typed | undefined | non-empty | Runs; replaces **title only** (branch preserved) |
| undefined | typed | empty | Skipped |
| undefined | typed | non-empty | Runs; replaces **branch only** (title preserved) |
| undefined | undefined | empty | Skipped — workspace name stays as the friendly random |
| undefined | undefined | non-empty | Runs; replaces **both** title and branch |
| (PR path) | any | any | Skipped unconditionally |

### F. Idempotency / explicit-id behaviors

| # | Scenario | Outcome |
|---|---|---|
| F1 | Submit, mutation in flight, user clicks again before response | Modal closed after first submit (renderer-side). Second click does nothing. |
| F2 | Submit, network drop after host insert but before client got response, retry with same `id` | Cloud transaction returns existing row by id, no new worktree, no duplicate. |
| F3 | Submit twice with same `(project, branch)` from different sessions | Second call sees existing local workspace row → `alreadyExists: true`. |
| F4 | Submit with explicit `id` matching existing in same org | Returns existing row (idempotent). |
| F5 | Submit with explicit `id` matching existing in **different** org | Cloud rejects CONFLICT. Worktree rolled back. |
